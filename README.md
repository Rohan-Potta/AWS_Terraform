# AWS Terraform Learning Repository

This repository contains a series of Terraform projects demonstrating Infrastructure as Code (IaC) practices for AWS services. It includes hands-on examples for provisioning and managing cloud resources such as S3 buckets, VPCs, EC2 instances, IAM users, and more.

## Projects

- **Project 01**: Hosting a static website on S3 with CloudFront distribution for caching and secure access.
- **Project 02**: VPC Peering between two VPCs in different AWS regions, including EC2 instances and routing.
- **Project 03**: Bulk IAM user management, creating users from CSV data and assigning them to groups based on roles.
- **Project 04**: Blue-Green deployment setup with application versions.
- **Project 05**: AWS Lambda function deployment with Python code.
- **Project 06**: Custom Terraform modules for reusable infrastructure components.
- **Project 07**: IAM configurations and policies.
- **Project 08**: Modular infrastructure with separate modules.
- **Project 09**: Lambda functions with modules.
- **Project 10**: Load balancer, auto-scaling groups, VPC, and security groups.
- **Project 11**: Multi-environment configurations (dev, test, prod) with infrastructure components.
- **Project 12**: Docker-based local deployment with frontend and backend.
- **Project 13**: Kubernetes manifests and Terraform infrastructure.

## Notes

The `notes/` directory contains educational materials on Terraform concepts, including:

- Introduction to Infrastructure as Code and its advantages
- Basic Terraform commands (init, validate, plan, apply, destroy)
- Terraform providers, versioning, and configuration
- State file management and common issues
- Variables, data types (primitives, lists, sets, maps, tuples), and precedence
- Recommended project structures and organization patterns
